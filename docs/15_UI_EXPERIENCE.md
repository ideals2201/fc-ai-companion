# 界面体验与物理驾驶舱设计

## 界面目标

界面不是普通网页面板，而是 FC 陪玩驾驶舱。
玩家应该直觉上看到：

- 左侧是 1P 实体手柄舱。
- 中间是电视。
- 电视下方是主机。
- 右侧是 2P 实体手柄舱。
- 下方是调试和数据显化区域。

## 主机设计

主机只负责真实主机相关功能：

- 更换卡带。
- 暂停/继续。
- Reset。

主机不放 Start。
Start 属于 1P / 2P 手柄。

主机不放音量、全屏、颜色。
这些属于电视。

## 卡带库设计

卡带库是玩家选择 ROM 的入口。

当前规则：

- 默认扫描 `D:\Ai-Play\ROM`。
- 玩家可以通过界面选择自己的 ROM 目录。
- 左侧显示 ROM 列表。
- 右侧显示选中 ROM 信息。
- 点击“更换卡带”后，才真正加载到模拟器。

卡带信息显示分层：

- 选中卡带详情：显示完整 ROM 信息。
- 当前插入：只显示简短状态，不重复完整详情。

当前插入行应显示：

- 中文名。
- ROM 显示名。
- ROM Profile。
- 策略完成状态。
- 适配状态。

## 电视设计

电视负责显示游戏画面和运行指标：

- 游戏画面。
- FPS。
- 帧数。
- 屏幕数。
- 全屏。
- 音量。
- 亮度。
- 对比。
- 色彩。

帧数、存活帧、屏幕数不应在 1P / 2P 或主机区域重复显示。

## 手柄舱设计

1P / 2P 手柄舱负责：

- 当前身份：玩家、AI 伙伴、混合。
- 人类 / AI 双开关。
- 策略按钮。
- 实体按键状态。
- 该玩家相关战斗数据。
- AI 对话。
- 数据流。

AI 未启用时，策略按钮不应显示选中状态。

## 数据显化原则

数据要紧凑、美观、可扫描。

优先显示：

- 击杀/摧毁分类。
- 死亡。
- 开枪。
- 实际发弹。
- 跳跃。
- 移动。
- 武器获得。
- 当前武器。
- 危险数量。
- 当前路线段。
- AI 写入状态。

技术字段可以显示，但必须服务玩家理解和调试，不应堆满界面。
