# 策略系统目标与验收标准

## 总目标

策略系统的目标不是让 AI 随机玩，而是让不同策略可以在真实 ROM、真实 RAM、真实手柄输入下完成可验证的通关行为。

## 三个核心验收目标

1. 单 AI 通关  
   一个 AI 可以按照指定策略，独立完成关卡通关。

2. 人类 + AI 配合通关  
   AI 能根据人类玩家状态配合完成通关，包括跟随、掩护、让路、补枪、避险和 Boss 集火。

3. 双 AI 协作通关  
   两个 AI 可以按不同分工互相配合完成通关，例如 1P 稳健推进、2P 护卫清敌。

## 策略类型

- 稳健生存：默认策略，优先活下来。
- 快速推进：优先向右推进，保留基础避险。
- 清敌优先：优先处理屏幕威胁。
- 奖励优先：优先武器箱、飞行胶囊和强化资源。
- 护卫队友：优先保护另一名玩家。
- 个人策略：玩家本地保存、训练或编辑的路线策略。

- 已完成真实手柄写入。

- 已完成 WorldX 路线基础：`WorldX = CameraX + PlayerX`。
- 已完成 Route Script V0。
- 已完成 Danger Layer V0：近身威胁和弹道威胁触发安全动作。
- 已完成基础瞄准：射击时根据目标高度自动上/下瞄准。
- 已完成 Action Lock V0：短时保持跳跃、闪避、瞄准射击和移动，减少逐帧抖动。
- 已完成 FSM V0：每路玩家区分菜单、推进、避险、攻击、Boss、复活和协作等待状态，并进入数据流显示。
- 已接入 2P 坐标候选 RAM：`0x0335` / `0x031B`，用于双人局中分离 2P 决策坐标，但仍需真实双人局验证。
- 已完成护卫协作 V0：2P 护卫策略以 1P 周围威胁为锚点，并用 2P 候选坐标保持队形距离。
- 已完成策略文件 ROM 绑定：`gameId + romProfileId + compatibilityGroup`。

## 验收证据

每个策略通过时必须保存：

- ROM Profile。
- 策略文件版本。
- 起止帧。
- 死亡数。
- 击杀/摧毁统计。
- 武器获得统计。
- Route Segment 轨迹。
- 输入轨迹。
- RAM 关键状态。
- 是否单 AI、人类 + AI、双 AI。

没有这些证据，不算“策略已完成”。
