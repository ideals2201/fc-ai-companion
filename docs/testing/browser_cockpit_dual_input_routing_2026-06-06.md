# 浏览器驾驶舱双手柄输入路由测试 - 2026-06-06

## 范围

验证 V0.1.0 之后的输入层扩展：

- 1P / 2P 独立控制器写入。
- 每路人类、AI、混合模式切换。
- 键盘双人输入。
- 浏览器 Gamepad API 轮询。
- 不破坏 V0.1.0 的 1P 手动输入。

## 键盘映射

1P：

- 方向键：方向
- `Z`：B
- `X`：A
- `Enter`：开始
- `Shift`：选择

2P：

- `W/A/S/D`：方向
- `J`：B
- `K`：A
- `I`：开始
- `U`：选择

## Gamepad 映射

- Gamepad 0：1P
- Gamepad 1：2P
- D-pad：buttons `12/13/14/15`
- 左摇杆：axes `0/1`
- A：button `0`
- B：button `2`，button `1` 作为备用 B
- 选择：button `8`
- 开始：button `9`

## 模式语义

- 人类：只接受人类输入。
- AI：当前为安全占位，默认空输入。
- 混合：人类输入可写入，后续 AI 可补助；冲突时人类优先。

## 验收重点

- 1P 旧键位仍可操作。
- 2P 屏幕按钮可写入 2P。
- 2P 键盘映射可写入 2P。
- 模式切到 AI 后，人类输入不写入该路。
- 混合模式保留人类输入，为后续 AI assist 预留仲裁层。
